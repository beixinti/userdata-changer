# Chrome 个人资料迁移工具

本工具通过 Windows mlink 目录联接命令，将 C 盘的 Chrome 用户数据全量无损迁移至其他磁盘，释放 C 盘空间，同时不影响 Chrome 的正常使用与更新。

## 🌟 核心功能

- **智能路径识别**：自动定位 Chrome 默认安装路径，支持手动纠错。
- **无损数据迁移**：采用 `robocopy` 多线程技术，确保文件权限、属性完整保留。
- **软链接建立**：使用 `mklink /j` 实现物理重定向，浏览器感知不到路径变化。
- **一键原路恢复**：通过自动生成的 `.ini` 配置文件，随时可以将数据迁回 C 盘。
- **鲁棒性处理**：自动处理用户输入中的双引号、末尾斜杠以及特定的 `\Default` 路径后缀。

## 🚀 使用方法

### 1. 环境准备

- 本工具适用 Windows 7/10/11（建议使用Windows 10及以上）。
- **确保以管理员身份运行**（脚本亦会自动检测并提醒）。

### 2. 开始迁移

1. 彻底关闭 Chrome 浏览器（建议使用前在任务管理器中确认没有 `chrome.exe`）。
2. 运行 `runme.bat`。
3. 在主菜单输入  `1` 以开始迁移Chrome个人资料。
4. 根据提示选择目标路径（默认 D 盘或手动输入其他盘符）。
5. 等待进度条完成，提示“迁移成功”后即可重新打开浏览器。

### 3. 恢复还原

1. 运行脚本并进入主菜单，输入 `2` 以恢复迁移。
2. 脚本将自动寻找目标盘中的配置文件并识别原始 C 盘位置。
3. 确认后，脚本将删除软链接并将文件搬回 C 盘。

## ⚠️ 注意事项

> [!IMPORTANT]
>
> **操作前必读**
>
> - **关闭浏览器**：迁移过程中如果 Chrome 正在运行，会导致文件被占用，进而导致迁移失败。
> - **不要手动删除软链接**：如果你手动删除了 C 盘的“快捷方式”图标（联接点），会导致浏览器找不着数据。如需操作，请使用脚本的“恢复”功能。
> - **编码格式**：如需修改脚本源码，保存时必须选择 **ANSI** 编码，否则中文会出现乱码。

> [!NOTE]
>
> - 本工具不会移动 Chrome 安装程序（包括可执行文件）位置。
> - 由于使用的是 Windows 的系统级目录链接命令，`chrome://version/`中的个人资料路径不会更改。

## 📂 文件结构说明

- `runme.bat`：主程序。
- `chrome个人资料迁移.ini`：迁移完成后自动生成于**目标文件夹**（默认D盘）中，记录了原始路径信息，如果您需要恢复，**请勿手动删除**。

## 🛠️ 技术原理

1. 验证管理员权限。
2. 使用 `robocopy "源路径" "目标路径" /E /MOVE` 移动数据。
3. 执行 `mklink /j "源路径" "目标路径"` 建立目录联接。
4. 写入配置文件以备后续溯源。
